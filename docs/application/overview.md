# Application Layer

## Purpose

The public manual provides reusable principles, systems, and programming rules.
The Application Layer helps one surfer use those systems without publishing
personal information.

> **The public manual teaches the system. The private workspace applies the system.**

Most users can read the manual online and use only the Starter Kit for personal
planning. You do not need to clone the manual, understand Git, or manage a local
documentation site.

## Choose a Setup Path

### 1. AI Project Workspace

**Recommended for most users.**

Create a private project-style workspace in ChatGPT, Claude, or a similar tool.
Add the Starter Kit templates, paste `PROJECT_INSTRUCTIONS.md` into the project's
instructions, and use the assistant to fill in personal context and maintain the
plan.

[Create an AI Project](create-ai-project.md)

### 2. Local Private Folder

**Best for users who want files on their computer.**

Copy the Starter Kit into a separate `Surf Athlete Personal/` folder. Edit the
Markdown files directly or open the folder with a file-aware assistant.

[Create a Local Private Workspace](create-private-workspace.md)

### 3. Git / Developer Workflow

**Best for technical users who want version history and file automation.**

Keep the personal workspace in a separate private repository or verified private
location. Do not place private files in the public manual repository unless you
understand Git history, exclusions, and the consequences of a mistake.

## Recommended Journey

**Start Here -> understand the system -> create a private AI Project -> add the
Starter Kit -> personalize with AI -> train, log, review, and adjust.**

1. Read the public manual online.
2. Create a private AI Project named `Surf Athlete Personal`.
3. Add or copy the Starter Kit files or templates.
4. Paste `PROJECT_INSTRUCTIONS.md` into the project instructions.
5. Fill in personal context with the AI assistant.
6. Generate and review the first training block.
7. Train, log, review, and adjust.

# Create an AI Project

## Recommended Setup

For most users, the easiest private application layer is a project-style
workspace in ChatGPT, Claude, or a similar AI tool. A project keeps its
instructions, reference material, and conversations together without requiring
Git or local Markdown tools.

Name the project:

```text
Surf Athlete Personal
```

Before adding personal information, check the project's visibility and sharing
settings. Keep it unshared unless you deliberately choose otherwise.

## Setup

1. Create a new project-style workspace in the AI tool.
2. Name it `Surf Athlete Personal`.
3. Open `starter-kit/PROJECT_INSTRUCTIONS.md` from the public manual.
4. Copy its full contents into the AI project's project instructions.
5. Add the Starter Kit files listed below as project files or knowledge.
6. Start a project conversation and ask the assistant to help complete
   `GOALS.md`, `BASELINE.md`, and `RECOVERY.md`.
7. Review the saved personal context before asking for the first training block.

## Add These Templates

- `GOALS.md`
- `BASELINE.md`
- `CURRENT_BLOCK.md`
- `TRAINING_LOG.md`
- `NUTRITION.md`
- `RECOVERY.md`
- `TRIPS.md`
- `EQUIPMENT.md`
- `NOTES.md`
- `HISTORY.md`
- `blocks/README.md`, when the platform supports folders or file paths

`README.md` is useful as an orientation page. `AGENTS.md` is intended for
file-aware coding agents and is not required in a normal AI Project.

## When the Platform Does Not Support Files

Paste each template into a separate project note, document, canvas, or message.
Use the template filename as the title so the assistant can distinguish goals,
baseline, current plan, logs, and history.

If the platform cannot maintain separate documents reliably, keep the Starter
Kit in one private document outside the public manual and paste only the relevant
section into the project conversation.

## Give the Assistant the Public Manual

The AI project is the private application layer. It still needs the public manual
as the source of principles.

Depending on the platform, provide the manual by:

- Linking to its public website when the assistant can read links
- Adding only the relevant public pages as reference material
- Pasting the relevant public protocol into the current conversation

Do not upload a private workspace into the public manual or treat an AI-generated
answer as a replacement for the manual's source and uncertainty rules.

## Privacy Check

Before entering dates, locations, measurements, health notes, or training logs:

- Confirm the project is not shared.
- Review the platform's data controls and retention settings.
- Understand who can access projects in a work, school, or team account.
- Avoid public share links and shared exports.
- Remove personal files before sharing a project or conversation.

See [Privacy](privacy.md) for the full checklist.

# Private Workspace Agent Instructions

## Scope

This file is for file-aware coding agents such as Codex, Claude Code, or similar
tools working directly with the copied private Starter Kit files.

For ChatGPT Projects, Claude Projects, or another project-style UI, use
`PROJECT_INSTRUCTIONS.md` as the persistent project instructions.

## First Read

1. Read `PROJECT_INSTRUCTIONS.md`.
2. Read the relevant public Surf Athlete Manual pages.
3. Read the private files needed for the current decision.

## File Boundary

- Treat this workspace as private.
- Use the public manual for principles and canonical protocols.
- Use these files for personal context and application.
- Write personal data only inside this private workspace.
- Never edit the public manual with private dates, locations, measurements,
  health context, logs, benchmarks, goals, or plans.
- Link to public protocols instead of copying full instructions.

## File Ownership

- `GOALS.md`: direction and success criteria
- `BASELINE.md`: facts, estimates, constraints, and unknowns
- `CURRENT_BLOCK.md`: current plan and adjustment rules
- `TRAINING_LOG.md`: completed work and response
- `NUTRITION.md`: nutrition context and observations
- `RECOVERY.md`: readiness and recovery rules
- `TRIPS.md`: private dates, locations, and travel demands
- `EQUIPMENT.md`: available tools and access
- `NOTES.md`: questions and temporary context
- `HISTORY.md`: durable lessons from completed blocks
- `blocks/`: archived plans and reviews

## Working Method

1. Ask only questions required for the current decision.
2. Use information already recorded before asking again.
3. Separate facts, athlete reports, assumptions, and unknowns.
4. Answer the narrow question first and explain the surf relevance.
5. Update only the private file that owns the information.
6. Show proposed changes before broad or multi-file edits.

## Safety and Recovery

- Follow `PROJECT_INSTRUCTIONS.md` and the public manual's recovery rules.
- Do not diagnose injuries, illnesses, or medical conditions.
- Stop or modify training for concerning pain, altered mechanics, illness, or
  symptoms that make the planned work unsafe.
- Recommend qualified evaluation when symptoms require it.
- Never use a personal record or planned volume to override safety.

## Update Discipline

- Update `CURRENT_BLOCK.md` when the active plan changes.
- Append completed work to `TRAINING_LOG.md`; do not silently rewrite history.
- Add durable lessons to `HISTORY.md` after a block review.
- Archive completed plans in `blocks/`.
- Avoid duplicating the same personal fact across files.
- Keep edits concise, direct, and easy to review.

## Purpose

The root `starter-kit/` directory contains blank Markdown templates for both AI
Project users and local file or folder users. It is a starting structure, not a
completed training plan, medical advice, or a substitute for assessment.

AI Project users can upload the templates, paste them into separate project
notes, or keep them in another private document. Local users copy the directory
into `Surf Athlete Personal/`.

## Download

[Download the Starter Kit](/downloads/surf-athlete-starter-kit.zip)

The website also publishes each blank template as a plain Markdown file under
`/starter-kit/`. The ZIP is generated from the canonical public `starter-kit/`
directory during every site build; it does not contain personal data.

## Files

- `README.md`: explains the three setup paths and recommended order of use.
- `PROJECT_INSTRUCTIONS.md`: copy/paste instructions for ChatGPT Projects, Claude
  Projects, or a similar project-style workspace.
- `AGENTS.md`: instructions for file-aware coding agents working directly with
  the private files.
- `GOALS.md`: records the next performance opportunity, long-term direction, and
  success criteria.
- `BASELINE.md`: records current movement, strength, conditioning, paddling, and
  recovery context.
- `CURRENT_BLOCK.md`: defines the active training block, weekly structure,
  progression, and adjustment rules.
- `TRAINING_LOG.md`: stores dated surf, paddle, training, and recovery entries.
- `NUTRITION.md`: records nutrition objectives, routines, constraints, and
  observations.
- `RECOVERY.md`: records readiness signals, recovery practices, and rules for
  modifying training.
- `TRIPS.md`: records private dates, locations, travel demands, and preparation
  requirements.
- `EQUIPMENT.md`: records available equipment and access constraints.
- `NOTES.md`: holds questions and personal context not yet owned elsewhere.
- `HISTORY.md`: summarizes completed blocks, major changes, and lessons learned.
- `blocks/README.md`: explains how to archive completed blocks without cluttering
  the current plan.

## Use

Keep the public templates blank and reusable. Fill in only private copies or
private AI Project notes. Link personal plans to the public manual's canonical
systems and exercises instead of copying full protocols.

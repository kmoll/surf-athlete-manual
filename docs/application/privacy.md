# Privacy

## Keep Personal Training Data Private

Do not publish personal training data in the public manual, a public repository,
or a public website.

Private data includes:

- Trip dates
- Locations
- Body weight and measurements
- Health and symptom notes
- Training logs
- Strength numbers and other benchmarks
- Recovery notes
- Nutrition notes
- Personal goals

For most users, use a private AI Project named `Surf Athlete Personal`. If you
prefer files on your computer, use a separate local `Surf Athlete Personal/`
folder. Do not put completed private Starter Kit files inside the public manual
repository.

## AI Project Privacy

An AI Project may contain uploaded files, saved instructions, conversation
history, and generated content. Before adding personal data:

- Confirm the project is not shared.
- Review the platform's privacy, retention, and data-control settings.
- Check who can access projects in a work, school, team, or family account.
- Understand whether project files or conversations can be shared by link.
- Remove private context before sharing a project, conversation, or export.

Project features and account policies vary. A project-style interface does not
make information private automatically.

## Other Exposure Paths

Personal information can also be exposed through:

- Uploaded files with broader permissions than expected
- Shared local or cloud folders
- Public Git repositories, commits, branches, or pull requests
- Generated PDFs that combine public and private pages
- Zip files containing both manual and personal folders
- Screenshots, exports, backups, or AI conversation sharing

Before publishing or sharing an artifact, inspect its contents, metadata, access
settings, and destination.

## Developer Warning

Technical users may use a separate private Git repository. Avoid storing private
files in the public manual repository. An ignored in-repository folder is an
advanced option with more ways to fail, and `.gitignore` cannot erase content
that has already entered Git history.

# AI Assistant Prompts

These prompts work with ChatGPT, Claude, Codex, or a similar assistant. Replace
bracketed placeholders only when needed. Keep all personal output in the private
AI Project or local workspace.

## 1. Set Up an AI Project

```text
Help me set up a private AI Project named "Surf Athlete Personal" using the
public Surf Athlete Manual and its Starter Kit.

Treat the public manual as the source of principles and the AI Project as the
private application layer. Tell me where to paste PROJECT_INSTRUCTIONS.md and
which Starter Kit templates to add as files, knowledge, or separate notes. Ask
only questions required for setup. Do not ask for personal training information
until I confirm the project is not shared and I have reviewed its privacy and
data settings. Do not place private data in the public manual.
```

## 2. Create a Local Folder Workspace

```text
Help me create a local private workspace named "Surf Athlete Personal" next to,
not inside, "Surf Athlete Manual". Copy the public Starter Kit structure into the
private folder.

Use the public manual for principles and the private folder for all personal
context and generated plans. Explain proposed file changes, preserve the
template structure, and do not invent personal details. This is a local-folder
workflow, not a request to add private files to the public repository.
```

## 3. Fill In Goals and Baseline

```text
Use the public manual for principles and my private workspace for personal
context. Help me complete GOALS.md and BASELINE.md.

Ask only unanswered questions that materially affect planning. Separate the next
surf opportunity from lifelong development. Record facts, estimates, and unknowns
clearly. Keep all answers in the private workspace. Do not create a training
program yet, diagnose medical issues, or invent benchmarks.
```

## 4. Create the First Training Block

```text
Read the public manual, then read GOALS.md, BASELINE.md, RECOVERY.md, TRIPS.md,
and EQUIPMENT.md in my private workspace. Propose the first recoverable training
block in CURRENT_BLOCK.md.

Use public systems and link to their canonical protocols. Explain why each
session exists, keep paddling the primary sport-specific conditioning method,
avoid training to failure, and include progression, regression, monitoring,
deload, and stop-or-modify rules. Ask only questions that block a safe decision.
State uncertainty and do not add volume merely to make the plan harder.
```

## 5. Review a Training Week

```text
Review the latest TRAINING_LOG.md entries against CURRENT_BLOCK.md, RECOVERY.md,
and my goals. Summarize what was planned, what was completed, how I responded,
and what remains uncertain.

Recommend only the smallest useful adjustment for the next week. Preserve an
additional hard session in reserve when practical. Update only the private
workspace, and distinguish general exercise physiology from approved expert
attribution.
```

## 6. Adjust for Recovery

```text
Use my recent TRAINING_LOG.md and RECOVERY.md entries to evaluate whether the
current plan is recoverable. Check sleep, soreness, pain or symptoms, motivation,
performance trend, surf and paddle volume, illness, travel, and accumulated
fatigue.

Choose whether to continue, reduce, replace, or stop the next planned session.
Explain the decision, favor adaptation over fatigue, and update CURRENT_BLOCK.md
or TRAINING_LOG.md as appropriate. Do not diagnose a condition. Flag symptoms
that need qualified medical evaluation and ask only questions necessary for the
immediate decision.
```
